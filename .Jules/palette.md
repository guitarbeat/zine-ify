## 2024-05-15 - Enhancing UX Feedback & Accessibility
**Learning:** Adding loading states (like text and icon changes during PDF generation) prevents users from repeatedly clicking disabled buttons. Furthermore, adding high-contrast, visually distinct focus indicators matching the app's aesthetic (punk theme) drastically improves keyboard accessibility while keeping the visual identity intact. Using distinct background colors for status tags (success/error/info) enhances at-a-glance readability over just changing text colors.
**Action:** Always provide explicit visual feedback (like disabled states and loading text/icons) for asynchronous operations, especially those that take a few seconds like PDF export. For themed applications, ensure focus indicators are present, high-contrast, and match the design system (e.g., using `focus-visible` to apply dashed outlines and yellow backgrounds).
