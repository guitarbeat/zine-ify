## 2024-05-18 - DOM XSS via innerHTML template interpolation
**Vulnerability:** File names uploaded by the user were interpolated directly into a template string and assigned to `.innerHTML` in `updateUploadedFilesList` within `src/js/zine-ui.js`. This allowed execution of malicious scripts (DOM XSS) when processing untrusted PDF file names (e.g. `<img src=x onerror=alert("XSS")>.pdf`).
**Learning:** Even elements dynamically generated in the client side using `.innerHTML` are vulnerable to Cross-Site Scripting (XSS) if any part of the interpolated string contains untrusted user input, such as a file name.
**Prevention:** Construct static HTML scaffolds using `.innerHTML` or `document.createElement`, and then securely inject dynamic user-controlled values into those structures using `.textContent` (or properties like `.value`). Do not directly interpolate dynamic user inputs into strings passed to `.innerHTML`.
